Internet Connection

The Stealth Browser can connect to a website through direct internet access and peer-to-peer via other machines that have internet access.

If this machine does not have internet access, select the peer-to-peer option. Remember to add the machine with internet access to the list of known Peers.

Internet History

The Stealth Browser can remember the internet browsing history. The history is remembered per Peer and is kept until the below condition is matched. It can be managed in detail at stealth:history.

Internet User-Agent

The Stealth Browser can emulate the User-Agent identifier of other Web Browsers to keep websites working while still being undetected. The User-Agent identifier is regenerated when a Tab is opened and it will be kept until the Tab is closed.

Hosts

The Stealth Browser has its own persistent DNS cache in order to prevent tracking capabilities of Internet Service Providers when being online via public Wi-Fi or 2G/3G/4G. Already cached websites and their resources will continue to work if their DNS entries are deleted.

Domain Hosts (IPv4/IPv6)

Peers

The Stealth Browser can share cached websites and resources among a list of trusted Peers. The connection represents the type of internet access the Peer has.

If a Peer is only reachable via a local hostname, remember to add its IP to the /etc/hosts file (and restart Stealth afterwards) to ensure a working connection. This ensures that Peers contain a valid Hosts entry, even when the Stealth Profile is deleted.

Domain Connection

Sites

The Stealth Browser does not request anything by default, and strictly follows the concept of establishing trust by whitelisting specific resources, and it does not trust anything by default.

Sites in Stealth are therefore configured in three ways:

  • Modes - allow URLs by media type
  • Beacons - extract Knowledge by queries

If you do not trust a Site anymore, remove its Modes. The Cache continues to persist and to work for this Stealth Service instance and all connected Peers.
